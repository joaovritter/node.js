# O que é Node.js

Node.js é um ambiente de execução que permite rodar JavaScript fora do navegador. Ele é baseado no motor V8 (o mesmo do Google Chrome) e é amplamente utilizado para desenvolver aplicações de backend, APIs e serviços web.

---

## Principais características do Node.js

- **Assíncrono e não bloqueante**  
  Opera principalmente de forma assíncrona, permitindo lidar com várias requisições ao mesmo tempo sem bloquear a execução do programa.

- **Single-thread com Event Loop**  
  Utiliza uma única thread principal, mas consegue alta performance graças ao *Event Loop*, que gerencia operações assíncronas.

- **Alta performance**  
  O uso do motor V8 compila JavaScript para código de máquina, tornando a execução rápida.

- **Orientado a eventos**  
  A execução do código é baseada em eventos, como requisições HTTP, leitura de arquivos e respostas do banco de dados.

- **Ecossistema rico (NPM)**  
  Possui o NPM (Node Package Manager), um dos maiores repositórios de bibliotecas do mundo.

- **Multiplataforma**  
  Funciona em Windows, Linux e macOS.

---

## Para que o Node.js é usado

- Criação de **APIs REST**
- Aplicações **backend**
- Sistemas em **tempo real** (chat, streaming, jogos online)
- Ferramentas de linha de comando (CLI)
- Microsserviços

---

## Termos técnicos importantes

- **V8 Engine**  
  Motor JavaScript criado pelo Google que executa e compila o código.

- **Event Loop**  
  Mecanismo responsável por gerenciar tarefas assíncronas e callbacks.

- **Callback**  
  Função passada como parâmetro para ser executada após uma operação terminar.

- **Promise**  
  Objeto que representa o resultado futuro de uma operação assíncrona.

- **Async / Await**  
  Sintaxe que facilita o uso de código assíncrono, deixando-o mais legível.

- **NPM (Node Package Manager)**  
  Gerenciador de pacotes usado para instalar bibliotecas e dependências.

- **Modules**  
  Forma de organizar o código em arquivos reutilizáveis (`require` / `import`).

- **API REST**  
  Interface que permite comunicação entre sistemas usando HTTP.

- **Middleware**  
  Funções intermediárias que processam requisições antes da resposta final.
